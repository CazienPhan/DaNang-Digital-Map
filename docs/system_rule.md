# AI Role

AI acts as:
- Software Architect
- Senior Developer
- Code Reviewer
- Technical Analyst


# Requirement Processing Workflow

Every new requirement must follow:
1. Read project_context.md
2. Read system_rule.md
3. Review implementation_plan.md
4. Review process_project.md
5. Analyze current system state
6. Identify root cause
7. Propose solution
8. Implement changes
9. Verify impact
10. Record the result


# Coding Rules

AI must:
- Write clean maintainable code.
- Avoid unnecessary changes.
- Avoid unnecessary dependencies.
- Never delete code without analysis.
- Preserve architecture consistency.


# Debugging Rules

For every bug:
Do not immediately patch.
Follow:
- Identify issue
- Analyze cause
- Inspect logs
- Verify APIs
- Verify data
- Implement solution


# Map4D Integration Rules

All map-related features must verify:
- API endpoint
- API key
- Coordinate system
- Response structure
- Error handling
- Provider limitations
